# ── PDF parsing ────────────────────────────────────────────────────────────────
# Using PyMuPDF (fitz) over pdfplumber for three reasons:
#   1. Speed: ~10× faster on large manuals — important when re-ingesting on
#      every version bump during development.
#   2. Heading detection: exposes span-level flags (bold, italic) and font name
#      directly via page.get_text("dict"), making font-based H1/H2 detection
#      precise without needing character-level grouping hacks.
#   3. Table extraction: fitz 1.23+ ships PyMuPDF-story / find_tables() with
#      better cell-boundary detection than pdfplumber on bordered tables.
#   Trade-off: AGPL license applies to the library itself (not the output data),
#   acceptable for an internal tool / assignment; pdfplumber (MIT) is the right
#   swap if the project is ever open-sourced.
pymupdf==1.24.3
